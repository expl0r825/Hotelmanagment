# Как да видите грешките при компилация

## В Visual Studio:

### Метод 1: Error List панел
1. Отворете **View → Error List** (или натиснете **Ctrl+W, E**)
2. Ще видите всички грешки, предупреждения и съобщения
3. Кликнете на всяка грешка, за да отидете до файла и реда

### Метод 2: Output панел
1. Отворете **View → Output** (или натиснете **Ctrl+Alt+O**)
2. В падащото меню изберете **Build**
3. Ще видите подробни съобщения за компилацията

### Метод 3: Build Output
1. Натиснете **Ctrl+Shift+B** за компилиране
2. В долната част на Visual Studio ще се появи панел с резултатите
3. Червените съобщения са грешки

## Често срещани грешки и решения:

### 1. "The type or namespace name 'X' could not be found"
**Решение:** Добавете липсващия using statement в началото на файла

### 2. "'X' does not exist in the current context"
**Решение:** 
- Проверете дали името е правилно написано
- Проверете дали имате правилните using statements
- Проверете дали класът/методът съществува

### 3. "Cannot convert from 'X' to 'Y'"
**Решение:** Проверете типовете данни - може да има несъответствие

### 4. "CS0246: The type or namespace name could not be found"
**Решение:** 
- Проверете дали пакетът е инсталиран
- Изпълнете `dotnet restore`

## Какво да направите сега:

1. **Отворете Error List** (Ctrl+W, E)
2. **Копирайте първите 3-5 грешки** и ги изпратете
3. Или **снимка на Error List панела**

Така ще мога да видя точно какви са проблемите и да ги поправя!
