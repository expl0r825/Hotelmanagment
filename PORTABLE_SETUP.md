# Направени промени за портативност

## Какво е направено

Проектът е направен напълно портативен и ще работи на всеки компютър след изтегляне от GitHub.

### 1. Конфигурационни файлове

✅ **Създадени примерни файлове:**
- `appsettings.Development.json.example` - шаблон за локална разработка
- `appsettings.Production.json.example` - шаблон за production

✅ **Обновен `.gitignore`:**
- Игнорира `appsettings.Development.json` и други user-specific файлове
- Позволява само примерните файлове да се комитират

### 2. Премахнати хардкодирани пътища

✅ **Обновени файлове:**
- `SETUP_DATABASE.md` - премахнат абсолютен път
- Всички пътища са относителни

### 3. Автоматична настройка

✅ **Обновен `start.bat`:**
- Автоматично създава `appsettings.Development.json` ако липсва
- Копира от примерния файл

### 4. Документация

✅ **Създадени нови файлове:**
- `SETUP_GUIDE.md` - пълно ръководство за настройка
- `CONTRIBUTING.md` - ръководство за сътрудничество
- `PORTABLE_SETUP.md` - този файл

✅ **Обновен `README.md`:**
- Добавени инструкции за първоначална настройка
- Връзки към подробни ръководства

## Как да използвате проекта след изтегляне

### Бърз старт:

1. **Изтеглете проекта:**
   ```bash
   git clone <repository-url>
   cd HotelManagement
   ```

2. **Стартирайте `start.bat`:**
   - Скриптът автоматично ще:
     - Създаде `appsettings.Development.json` (ако липсва)
     - Възстанови пакетите
     - Компилира проекта
     - Създаде базата данни
     - Стартира приложението

3. **Или ръчно:**
   ```bash
   copy appsettings.Development.json.example appsettings.Development.json
   dotnet restore
   dotnet ef migrations add InitialCreate
   dotnet ef database update
   dotnet run
   ```

## Важни бележки

### Файлове, които НЕ се комитират:
- `appsettings.Development.json` - всеки разработчик има свой
- `appsettings.Production.json` - production конфигурация
- `bin/` и `obj/` - компилирани файлове
- `.vs/` - Visual Studio настройки

### Файлове, които СЕ комитират:
- `appsettings.json` - основна конфигурация (без чувствителни данни)
- `appsettings.*.example` - шаблони за копиране
- Всички изходни файлове (`.cs`, `.cshtml`, и т.н.)
- `wwwroot/images/` - статични снимки на стаи

## Проверка на портативността

За да проверите дали проектът е портативен:

1. Изтеглете проекта на друг компютър
2. Изпълнете `start.bat`
3. Приложението трябва да работи без промени

Ако има проблеми, проверете:
- Да имате .NET 8.0 SDK
- Да имате SQL Server LocalDB
- Да сте копирали `appsettings.Development.json`

## Поддръжка

Ако имате проблеми с настройката, вижте:
- `SETUP_GUIDE.md` - подробни инструкции
- `SETUP_DATABASE.md` - настройка на базата данни
- `README.md` - основна информация
